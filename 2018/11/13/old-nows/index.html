部分基本内容回顾

一、基本数据类型

1.1 number string boolean null undefined

二、复杂数据类型 对象（广义的）

2.1 Array Date Math new string（）

new Number（） new Boolean（） function

三、基本包装类型对象的概念

js为了方便开发者开发操作基本数据类型值，提供一个对象进行操作

四、基本数据类型和复杂数据类型的区别

4.1 保存数据的个数不同

4.2 在内存中保存方式不同

基本数据类型在内存单元中保存的是具体值

复杂数据类型在内存单元中保存的是具体值得指针（地址）

4.3 复制操作的结果不同

复制操作复制的内存单元中保存的值

基本数据类型复制前后，两个值没有关系

复杂数据类型复制前后，两个值是同一个值

4.4 函数参数值传递（复制）的结果不同

因为基本数据类型和复杂数据类型在复制后结果不同，作为函数传递后形参和实参的关系不同

小结：如果实参是基本数据类型，修改后必须设置返回值（获取型操作）

如果实参是复杂数据类型，修改后不需要设置返回值（设置型操作）

六、prototype——原型

6.1 每个函数都具有一个属性prototype，值为一个对象

这个属性其实是给构造函数准备的，其他的函数不使用这个属性的功能

作用：用来保存构造函数中所用需要公用的方法，简化代码书写。

6.2 为什么对象可以使用原型对象中的方法？

通过观察对象，我们发现对象具有一个proto属性，是new给对象设置的

通过比较发现，这个proto与.prototype实际上是一个值

6.3 对象的属性访问方式

首先找到自身，如果存在，使用。

如果没有找到，找到proto属性，如果有，使用

七、constructor—构造器

是prototype的属性，但是不是给自己准备的，而是让创建的对象使用的。

作用：用来描述对象和构造函数之间关系的一种方式

八、this的使用

普通函数中：window

方法中：当前对象（调用者）

原型的方法中：创建的对象

构造函数中：创建的对象

九、面向对象简介

9.1 基本概念：

面向：专注于某个事物

面向于对象：专注于对象的操作

面向于过程：专注于过程的操作

例如要对数值数组进行排序操作，这就是一种面向对象的操作方式。

使用数组对象—sort（）—调用方法完成功能 —传入参数进行降序排序

9.2 以前使用过的面向对象和面向过程操作：

面向过程：大部分代码都是面向过程的

面向对象：内置对象功能，jQuery功能，

面向对象的操作总体来说就是四个字：拿来主义。

9.3 封装：将功能封装成为对象，用于功能的复用。

继承：为了实现更好地复用。

多态：增强功能使用的灵活性。

9.4 js是基于原型的语言

js是基于对象的语言（通过大量的内置对象功能完成操作）

js原生语法中不支持面向对象操作，我们可以通过一些操作取模拟面向对象的特征，从而实现面向对象操作。
